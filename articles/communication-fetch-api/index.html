    探索 Fetch API

    • JavaScript
    • Web API
    • 跨源通信
    最后更新： 2018-01-05
    阅读次数： 次
    许可协议： 知识共享署名-非商业性使用-禁止演绎 4.0 国际许可协议

    XMLHttpRequest 已经存在了好长时间，我们常用它来执行异步请求，但是由于它在设计上不符合职责分离原则，将输入、输出和用事件来跟踪的状态混杂在一个对象里， 所以它不是最佳 API，Fetch API 的出现，旨在修复这些缺陷。

    • Fetch API 接口
      • GlobalFetch.fetch()： 用于发起获取资源的请求，返回一个 promise 对象
      • Headers 对象： 对 HTTP request 和 response headers 执行各种操作
      • Request 对象： 用来表示资源请求
      • Response 对象： 呈现了对一次请求的响应数据

    以上就是 Fetch 的核心 API。当然，上面列出的几个对象都有各自的属性和方法，这个我就先不一一展开了，以后有空了再补。（其实看 MDN 文档就已经足够了）
